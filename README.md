# 勤怠エラー文章作成ツール

## 概要
Touch On Time から出力した**日別勤怠CSVを読み込み**、指定期間内のエラー対象者を自動抽出して、Slack / LINE WORKS にそのまま貼り付けられる通知文を生成するツールです。

## 主な機能
- CSVファイルのアップロード（ドラッグ＆ドロップ対応）
- 対象期間の開始日・終了日を指定（両端の日を含む）
- 通知文の月表記・申請期限文言を入力で変更可能
- エラー条件に基づいた自動判定（10条件）
- 部署ごと・人名ごとに整理された文章を生成
- 全文コピーボタン付き
- エラー一覧をテーブル形式で確認可能

## エラー判定条件
1. 通常勤務なのに出退勤のいずれかまたは両方がない
2. 出勤打刻が重複している
3. 退勤打刻が重複している
4. スケジュールが「直帰」なのに退勤打刻がある
5. スケジュールが「直行」なのに出勤打刻がある
6. 休暇区分が「AM有休」なのに打刻不足
7. 休暇区分が「PM有休」なのに打刻不足
8. 法定休日・公休 全日・有休 全日などの全休日に打刻がある
9. 対象日の退勤打刻が対象日の翌日になっている
10. 直行直帰でも翌日退勤だけはエラーとして拾う

※ `勤務日種別=法定休日` と `休暇区分=公休 全日 / 有休 全日` のどちらでも全休として扱います。

## 対応CSVフォーマット
Touch On Time の日別勤怠データ出力CSV（150列前後の標準形式）に対応。  
文字コードは UTF-8（BOM有無どちらも可）・Shift-JIS・EUC-JP に自動対応。

### 使用する主な列（固定インデックスで読み取り）
| 列番号 | 内容 |
|--------|------|
| COL[3]  | 所属部署名 |
| COL[7]  | 氏名 |
| COL[30] | 勤務日（YYYY/M/D 形式） |
| COL[33] | パターン名（通常勤務/直帰/直行） |
| COL[34] | 勤務日種別（平日/法定休日） |
| COL[35] | 休暇区分（公休 全日 / 有休 全日 / AM有休 / PM有休） |
| COL[36] | 備考・スケジュール内容 |
| COL[45] | 出勤実績 |
| COL[46] | 退勤実績（翌日日付になる場合あり） |
| COL[51] | 出勤打刻（方法付き・重複検出に使用） |
| COL[52] | 退勤打刻（方法付き・重複検出に使用） |

## 日付フィルタリングの仕様
- 開始日・終了日ともに**当日を含む**
- 日付は `YYYYMMDD` 形式の整数に変換して比較（時刻による誤判定を完全に排除）
- 修正前の問題：終了日を 2026/4/16 に設定しても 4/30 までのデータが含まれていた → **修正済み**

## ファイル構成
- `index.html` — ツール本体（1ファイルで完結）
- `.github/workflows/deploy-pages.yml` — GitHub Pages 自動公開
- `.gitignore` — 実CSVのコミット防止
- `GITHUB_PAGES_SETUP.md` — 公開手順書

## 動作環境
- モダンブラウザ（Chrome / Firefox / Edge / Safari）
- インターネット接続不要（ブラウザ内で完結）
- Genspark アカウント不要

## GitHub Pages での公開
- GitHub Pages 用の workflow を同梱しています
- push 時に `index.html` だけを Pages にデプロイするため、実CSVは公開対象に入りません
- 公開手順は `GITHUB_PAGES_SETUP.md` を参照してください
